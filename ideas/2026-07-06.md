# 2026-07-06 오늘의 아이디어

## 1. 나가도 돼?
- 평가: (아직 없음)
- 유형: 새로움
- 불편: 폭염 경보가 일상이 된 여름, 러너·개 산책인·아이 키우는 부모가 매일 아침 "오늘 몇 시에 나가야 안 죽지?"를 날씨 앱 시간대별 화면을 뒤져가며 직접 계산한다.
- 타깃: 매일 밖에 나가야 하는 사람 (운동, 개 산책, 등하원)
- MVP 기능 (딱 하나): 오늘 하루를 시간대별 색깔 띠(초록=안전/노랑=주의/빨강=위험)로 보여주고 "다음 초록 구간: 저녁 7시"를 크게 표시
- 난이도: 쉬움
- 영감이 된 뉴스: 미국 인구 절반인 1.5억 명이 폭염 경보권 — [CNN](https://www.cnn.com/2026/07/01/climate/heat-waves-europe-us-el-nino-warming), [NPR 폭염 대처법](https://www.npr.org/2026/06/30/nx-s1-5874848/life-kit-how-to-handle-the-heat-wave) ("이른 아침이나 늦은 저녁에 운동하라"는 조언을 자동화)
- 자기평가: 8/10 — 지금 계절에 딱 맞고, 열자마자 색깔 띠 하나로 끝나는 10초 가치가 명확하다

### 빌드 프롬프트
> 브라우저 위치 권한으로 현재 위치를 받아 Open-Meteo API(키 불필요)에서 오늘의 시간별 체감온도(apparent_temperature)를 가져오는 단일 페이지 웹앱을 만들어줘.
> 화면에는 0~24시를 가로 색깔 띠로 표시: 체감온도 27°C 미만 초록, 27~32°C 노랑, 32°C 이상 빨강.
> 맨 위에 현재 상태("지금 나가면 위험 🥵")와 "다음 초록 구간: 19:00"을 크게 보여줘.
> 프레임워크 없이 HTML+CSS+바닐라 JS 파일 하나로, 모바일 우선으로.

## 2. 에어컨 요금 미리보기
- 평가: ✅(만들었음) — `apps/aircon-bill/`
- 유형: 새로움
- 불편: 폭염에 에어컨을 하루 종일 틀면서도 한국 가정집은 누진세 때문에 "이번 달 전기세 폭탄 아닐까"를 매일 불안해하지만, 고지서가 나올 때까지 한 달간 알 방법이 없다.
- 타깃: 여름에 에어컨 트는 모든 자취생·가정집
- MVP 기능 (딱 하나): 에어컨 소비전력(W)과 하루 사용 시간을 입력하면 주택용 누진 구간을 반영한 이번 달 예상 전기요금을 즉시 표시
- 난이도: 쉬움
- 영감이 된 뉴스: 2026년 여름 기록적 폭염으로 전력 소비 사상 최고치 경신 — [Matrix Mag](https://matrixmag.com/heatwaves-hot-summer-2026-global/), [CNN](https://www.cnn.com/2026/07/01/climate/heat-waves-europe-us-el-nino-warming)
- 자기평가: 7/10 — API 없이 요금표 하드코딩만으로 완성되고 불편이 아주 구체적이나, 누진 구간 계산 정확성 검증에 반나절은 써야 한다

### 빌드 프롬프트
> 한국 주택용(저압) 전기요금 누진제를 반영한 에어컨 전기요금 계산기 웹앱을 만들어줘.
> 입력: 에어컨 소비전력(W), 하루 평균 사용 시간, 월 기본 사용량(kWh, 기본값 300).
> 출력: 에어컨 추가분을 합친 이번 달 예상 요금과 "에어컨 없을 때 대비 +XX,XXX원"을 크게 표시.
> 2026년 기준 누진 구간(300kWh/450kWh 경계)을 코드에 상수로 넣고, 슬라이더를 움직이면 실시간으로 요금이 바뀌게 해줘.
> HTML+CSS+바닐라 JS 파일 하나, 저장은 localStorage.

## 3. 구독료 체감 변환기
- 평가: (아직 없음)
- 유형: 비틀기
- 불편: 구독 서비스가 월 몇천 원씩이라 무감각해진 직장인이 카드 명세서를 볼 때마다 "이걸 다 합치면 대체 얼마야?"라고 놀라지만, 연 단위 합계를 체감되는 값으로 보여주는 곳이 없다.
- 타깃: 구독 서비스 3개 이상 쓰는 직장인·대학생
- MVP 기능 (딱 하나): 구독 이름과 월 요금을 입력하면 연간 총액을 "치킨 23마리 / 제주도 왕복 2번 / 국밥 61그릇"처럼 체감 물가로 즉시 변환해서 표시
- 난이도: 쉬움
- 영감이 된 뉴스: "subscription creep"에 대한 사용자 반발 확산 — [The Cool Down](https://www.thecooldown.com/green-business/smartphone-app-subscription-issues-modern-economy/)
- 자기평가: 7/10 — 구독 관리 앱 자체는 흔하지만 "관리" 대신 "체감 충격 한 방"으로 좁힌 게 포인트, 공유하고 싶어지는 결과 화면이 강점

### 빌드 프롬프트
> 구독 서비스 이름과 월 요금을 여러 개 입력하면 연간 총액을 계산해주는 단일 페이지 웹앱을 만들어줘.
> 핵심은 결과 표시: 연간 총액을 "치킨 N마리(2.3만원), 국밥 N그릇(1.2만원), 제주도 왕복 N회(15만원)" 같은 체감 단위로 크게 보여줘.
> 항목별로 "이거 하나 끊으면 1년에 치킨 N마리" 배지도 붙여줘.
> 데이터는 localStorage에 저장하고, 결과 화면은 이미지로 저장해 공유하기 좋게 카드 형태로 디자인해줘.
> HTML+CSS+바닐라 JS 파일 하나로.
