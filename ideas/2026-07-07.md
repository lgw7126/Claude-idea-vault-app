# 2026-07-07 오늘의 아이디어

## 1. 택배 지연 보상금 계산기
- 평가: (아직 없음)
- 유형: 새로움
- 불편: 국지성 집중호우가 잦은 장마철(7월), 온라인 쇼핑을 자주 하는 사람이 배송이 며칠씩 늦어질 때마다 "택배가 늦으면 돈으로 보상받을 수 있다"는 규정(운송장 요금의 50%×지연일수)을 몰라서 그냥 넘어간다
- 타깃: 장마철에 택배 배송 지연을 겪는 온라인 쇼핑 이용자
- MVP 기능 (딱 하나): 인도예정일, 실제 수령일, 운송장 요금을 입력하면 표준약관 기준 보상 청구 가능 금액을 즉시 계산해서 보여준다
- 난이도: 쉬움
- 영감이 된 뉴스: 운송물 연착 시 초과일수×운송장 요금의 50%를 배상하도록 정한 표준약관 규정 — [찾기쉬운 생활법령정보](https://easylaw.go.kr/CSP/CnpClsMain.laf?popMenu=ov&csmSeq=663&ccfNo=3&cciNo=1&cnpClsNo=4), 2026년 장마철 국지성 호우로 배송 지연 우려 — [위기브](https://www.wegive.co.kr/wezine/detail/1690)
- 자기평가: 8/10 — 계산 근거가 실제 법령 조항이라 "근거 없는 계산기" 함정이 없고, 몰라서 포기하던 돈을 되찾아준다는 점이 명확하다. 감점 2는 실제 청구까지 이어지려면 사용자가 직접 고객센터에 전화해야 한다는 점(앱이 청구를 대신해주진 못함)

### 빌드 프롬프트
> 화면 1개짜리 웹앱을 만들어줘. 입력: 인도예정일(날짜), 실제 수령일(날짜), 운송장에 적힌 배송 요금(원).
> 계산: 지연일수 = 수령일 - 예정일, 보상금 = 운송장 요금 × 50% × 지연일수(단, 최대 운송장 요금 100% 한도).
> 결과를 "받을 수 있는 보상금: N원"으로 크게 보여주고, 아래에 계산 근거(표준약관 조항 요약)와 택배사 고객센터 연결 안내 문구를 작게 표시해.
> 로그인·DB·외부 API 없이 순수 계산만, HTML+CSS+바닐라 JS 파일 하나로, 모바일 우선.

## 2. 이거 상했나
- 평가: (아직 없음)
- 유형: 새로움
- 불편: 무더운 여름, 배달음식이나 도시락을 실온에 깜빡 두고 나갔다 온 사람이 거의 매일 "이거 먹어도 될까, 버려야 하나"를 감으로만 판단한다
- 타깃: 여름철 배달음식·도시락을 자주 시키는 자취생·직장인
- MVP 기능 (딱 하나): 음식 종류, 실온에 방치된 시간, 현재 기온을 입력하면 식품안전 가이드라인(32°C 이상 1시간, 상온 2시간 룰)에 따라 "안전 / 주의 / 폐기 권장" 판정을 보여준다
- 난이도: 쉬움
- 영감이 된 뉴스: 2026년 여름 기록적 폭염으로 실온 방치 식중독 위험 증가 — [Matrix Mag 폭염 보도](https://matrixmag.com/heatwaves-hot-summer-2026-global/)
- 자기평가: 8/10 — 판정 규칙이 공인된 식품안전 기준이라 근거가 명확하고, 열자마자 판정 하나로 끝나 10초 룰을 확실히 통과한다. 감점 2는 음식 종류별 세부 기준까지 정교화하려면 조사가 더 필요하다는 점

### 빌드 프롬프트
> 화면 1개짜리 웹앱을 만들어줘. 입력: 음식 종류(일반/유제품·해산물 중 선택), 실온에 둔 시간(시간 단위 슬라이더), 현재 기온(°C, 기본값은 브라우저 위치 기반 Open-Meteo API로 자동 조회).
> 판정 규칙: 32°C 이상이면 1시간, 그 미만이면 2시간을 기준으로 "안전(초록) / 주의(노랑) / 폐기 권장(빨강)"을 큰 텍스트와 색깔로 보여줘. 유제품·해산물은 기준 시간을 절반으로 적용.
> 로그인·DB 없이, 결과 아래에 판정 근거 한 줄을 작게 표시해. HTML+CSS+바닐라 JS 파일 하나, 모바일 우선.

## 3. 반품 마감일 트래커
- 평가: (아직 없음)
- 유형: 개선
- 불편: 온라인 쇼핑을 자주 하는 사람이 "마음에 안 들면 반품해야지" 하고 미루다가 반품 기한(보통 수령 후 7일)을 놓쳐 환불받을 돈을 날리는 일이 구매할 때마다 반복된다
- 타깃: 온라인 쇼핑을 자주 하는 1인 가구·직장인
- MVP 기능 (딱 하나): 상품명, 수령일, 반품 기한(일수, 기본 7일)을 입력하면 D-day를 큰 숫자로 보여주고 임박 순으로 목록 정렬
- 난이도: 쉬움
- 영감이 된 뉴스: (뉴스 기반이 아닌 반복 소비 경험 기반 — 온라인 쇼핑 반품 규정상 통상 7~14일 기한이 소비자에게 잘 인지되지 않는 구조적 문제)
- 자기평가: 6/10 — 불편 자체는 분명하고 돈이 걸린 문제지만, 푸시 알림 없이는 사용자가 스스로 앱을 열어봐야 알 수 있어 "까먹어서 놓친다"는 근본 원인을 완전히 해결하진 못한다. 그럼에도 목록으로 한눈에 모아 보여주는 것만으로도 지금 없는 가치를 준다

### 빌드 프롬프트
> 화면 1개짜리 웹앱을 만들어줘. 입력 폼: 상품명, 수령일(날짜), 반품 기한(일수, 기본값 7). 등록하면 목록에 추가.
> 목록은 반품 마감일까지 남은 일수(D-day)가 적은 순으로 정렬하고, D-2 이하는 빨간색으로 강조해.
> 데이터는 localStorage에 저장, 로그인·서버 없음. HTML+CSS+바닐라 JS 파일 하나로, 모바일 우선.
